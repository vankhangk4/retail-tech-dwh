Luồng khóa ngoại chi tiết (Foreign Key Flow)

DimDate
  PK: DateKey (INT, yyyyMMdd)
       │
       ├─→ FactSales.DateKey
       ├─→ FactInventory.DateKey
       └─→ FactPurchase.DateKey

DimProduct
  PK: ProductKey (INT, Surrogate)
  BK: ProductCode (VARCHAR)
       │
       ├─→ FactSales.ProductKey
       ├─→ FactInventory.ProductKey
       └─→ FactPurchase.ProductKey

DimCustomer
  PK: CustomerKey (INT, Surrogate)
  BK: CustomerCode (VARCHAR)
       └─→ FactSales.CustomerKey

DimStore
  PK: StoreKey (INT, Surrogate)
  BK: StoreCode (VARCHAR)
       ├─→ FactSales.StoreKey
       ├─→ FactInventory.StoreKey
       ├─→ FactPurchase.StoreKey
       └─→ DimEmployee.StoreKey

DimEmployee
  PK: EmployeeKey (INT, Surrogate)
  BK: EmployeeCode (VARCHAR)
       └─→ FactSales.EmployeeKey

DimSupplier
  PK: SupplierKey (INT, Surrogate)
  BK: SupplierCode (VARCHAR)
       ├─→ FactInventory.SupplierKey
       └─→ FactPurchase.SupplierKey

1.2. Lý do cần xây dựng Data Warehouse
    1.2.1. Tập trung hóa dữ liệu (SSOT)

    1.2.3. Nâng cao chất lượng dữ liệu

    2.1.2. Danh sách bên liên quan (Stakeholders)
    2.1.3. Danh sách câu hỏi phỏng vấn nghiệp vụ

2.2. Yêu cầu chức năng

CHƯƠNG 3: THIẾT KẾ KIẾN TRÚC HỆ THỐNG TỔNG THỂ

3.1. Lựa chọn kiến trúc và justification
    3.1.1. So sánh các mô hình kiến trúc (Monolithic, 3-tier, Data Lake, Lambda)
    3.1.2. Kiến trúc đề xuất: 4-Tier Layered Architecture

3.2. Mô tả chi tiết từng tầng
    3.2.1. Tầng 1: Data Sources (Nguồn dữ liệu)
        └── 3 nguồn: POS/Excel, Kho/Google Sheets, Danh mục CSV
    3.2.2. Tầng 2: ETL Processing Layer
        ├── Staging Database
        ├── Transformation Engine (Python / Stored Procedures)
        └── ETL Orchestrator (Python Scheduler / SQL Agent)
    3.2.3. Tầng 3: Data Warehouse (SQL Server)
        ├── Staging Layer (STG_*)
        ├── Core DWH Layer (Dim/Fact)
        └── Data Mart Layer (DM_*, Aggregate Tables)
    3.2.4. Tầng 4: BI & Visualization (Apache Superset)
        ├── Interactive Dashboards
        ├── Scheduled Reports
        └── RBAC (Role-based Access Control)

3.3. Luồng dữ liệu tổng thể (Data Flow Diagram - Level 1 & 2)

3.4. Công nghệ và Stack kỹ thuật
    ├── Database: SQL Server
    ├── ETL: Python (pandas, pyodbc), T-SQL (Stored Procedures)
    ├── Scheduling: Python APScheduler / SQL Server Agent
    ├── BI Platform: Apache Superset + SQLAlchemy
    ├── Version Control: Git
    └── OS: Linux / Windows

3.5. Sơ đồ triển khai (Deployment Diagram)
CHƯƠNG 4: THIẾT KẾ CƠ SỞ DỮ LIỆU

4.1. Phương pháp thiết kế
    4.1.1. Lý thuyết Dimensional Modeling (Ralph Kimball)
    4.1.2. Lựa chọn Star Schema — justification

4.2. Thiết kế bảng Dimension (chiều)
    4.2.1. DimDate — Chiều Thời gian (pre-populated)
        ├── Cấu trúc vật lý, các cột chi tiết
        └── Phương pháp khởi tạo (script)
    4.2.2. DimProduct — Chiều Sản phẩm (SCD Type 2)
        ├── Cấu trúc vật lý (ProductKey, ProductCode, ..., SCD columns)
        └── Chiến lược SCD Type 2 (EffectiveDate, ExpirationDate, IsCurrent)
    4.2.3. DimCustomer — Chiều Khách hàng
    4.2.4. DimStore — Chiều Cửa hàng
    4.2.5. DimEmployee — Chiều Nhân viên
    4.2.6. DimSupplier — Chiều Nhà cung cấp

4.3. Thiết kế bảng Fact (sự kiện)
    4.3.1. FactSales — Bán hàng (Transaction Fact)
        ├── Grain (độ hạt): 1 dòng = 1 sản phẩm / 1 hóa đơn
        ├── Cấu trúc vật lý
        └── Các measure: Quantity, GrossSalesAmount, NetSalesAmount, CostAmount, GrossProfitAmount
    4.3.2. FactInventory — Tồn kho (Periodic Snapshot Fact)
        ├── Grain: 1 dòng = 1 sản phẩm / 1 cửa hàng / 1 ngày
        └── Measures: OpeningStock, ClosingStock, TotalInventoryValue
    4.3.3. FactPurchase — Nhập hàng (Transaction Fact)

4.4. Thiết kế Staging Tables
    4.4.1. Danh sách các bảng Staging
    4.4.2. Ánh xạ từ nguồn vào Staging (Source → STG mapping)

4.5. Thiết kế bảng hệ thống
    4.5.1. ETL_Watermark (cơ chế trích xuất tăng dần)
    4.5.2. ETL_RunLog (kiểm toán pipeline)
    4.5.3. STG_ErrorLog (quản lý lỗi)

4.6. Thiết kế Data Mart (Aggregate Tables)
    4.6.1. DM_SalesDailySummary
    4.6.2. DM_InventoryAlert

4.7. Sơ đồ ERD Star Schema (PlantUML code)
    └── Trực quan hóa toàn bộ mô hình

4.8. Ràng buộc toàn vẹn và index strategy
    ├── Primary Key / Foreign Key constraints
    ├── Clustered Index vs Non-clustered Index
    └── Partitioning strategy (theo DateKey)

    5.1.2. Cơ chế điều phối (Orchestration)

5.2. Bước 1: EXTRACT (Trích xuất)

    5.2.2. Kết nối cơ sở dữ liệu nguồn (pyodbc)

5.3. Bước 2: TRANSFORM (Biến đổi)

5.4. Bước 3: LOAD (Nạp dữ liệu)
